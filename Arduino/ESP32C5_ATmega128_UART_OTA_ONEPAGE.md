# ESP32-C5 -> ATmega128 UART OTA 최종 회로도 한눈에 보기

전제:

- **최초 ISP 작업은 PC에서 진행**
- 보드에는 **UART OTA용 회로만 추가**

---

## 1. 최종 구조

```text
기존 유지:
ESP32 GPIO5  -------------------------------> ATmega RXD0
ESP32 GPIO4  <------------------------------- ATmega TXD0
ESP32 GND    -------------------------------- ATmega GND

새로 추가:
ESP32 GPIO24 --[2.2k]--> Base   Q1:2N3904
ESP32 GND ---------------------> Emitter
Collector ---------------------> ATmega RESET

ATmega RESET --[10k]-----------> +5V

선택:
Base --[100k]------------------> GND
```

---

## 2. 한 장으로 보는 회로도

```text
                 +---------------- ESP32-C5 ----------------+
                 |                                          |
                 | GPIO5  -------------------------------------> ATmega RXD0
                 | GPIO4  <------------------------------------- ATmega TXD0
                 | GND    -------------------------------------- ATmega GND
                 |                                              |
                 | GPIO24 ---[R1 2.2k]---B                      |
                 |                        Q1 2N3904             |
                 | GND -------------------E                     |
                 +------------------------C---------------------> ATmega RESET


                                                ATmega RESET ---[R2 10k]--- +5V

선택:
Q1 Base ---[R3 100k]--- GND
```

---

## 3. 지금 회로에서 바뀌는 것 / 안 바뀌는 것

### 그대로 두는 것

- `GPIO5 -> ATmega RXD0`
- `GPIO4 <- ATmega TXD0`
- `GND 공통`
- DWIN 회로
- SD 회로
- RGB / 버튼 / 전원 회로

### 새로 추가하는 것

- `GPIO24` 선 1가닥
- `2N3904` 1개
- `2.2k` 1개
- `10k` 1개
- 선택 `100k` 1개

---

## 4. 부품표

### 필수

- `Q1 = 2N3904`
- `R1 = 2.2k`
- `R2 = 10k`

### 선택

- `R3 = 100k`

---

## 5. 역할 요약

| 부품 | 역할 |
|---|---|
| `GPIO24` | ATmega 리셋 제어 신호 |
| `R1 2.2k` | 트랜지스터 Base 전류 제한 |
| `Q1 2N3904` | ATmega RESET을 GND로 당김 |
| `R2 10k` | RESET 풀업 |
| `R3 100k` | Base pulldown, 선택사항 |

---

## 6. 동작 방식

| GPIO24 상태 | 결과 |
|---|---|
| `LOW` | RESET 해제, ATmega 정상 동작 |
| `HIGH` | RESET Low 강제 |

즉 ESP32가 `GPIO24`를 잠깐 `HIGH`로 만들면 ATmega를 재시작시켜 UART 부트로더 진입을 시도할 수 있다.

---

## 7. 최종 결론

이번 프로젝트에서 보드에 실제로 추가할 회로는 이것뿐이다.

- **기존 UART 3선 유지**
- **RESET 제어 회로만 추가**
- **RESET용 캐패시터는 추가하지 않음**
- **SPI/ISP 회로는 보드 OTA용으로는 신경 쓰지 않음**

즉 한 줄로 끝내면:

> **`GPIO24 -> 2.2k -> 2N3904 -> ATmega RESET`, 그리고 `RESET -> 10k -> +5V`**
