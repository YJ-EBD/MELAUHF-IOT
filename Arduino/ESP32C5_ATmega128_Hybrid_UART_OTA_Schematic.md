# ESP32-C5 + ATmega128 하이브리드 OTA 회로도

목표 구조:

- 처음 1회: `ISP`로 `부트로더 + 앱 + 퓨즈` 기록
- 이후 현장 업데이트: `TX/RX/GND + RESET`으로 `UART OTA`

즉 이 회로는 `양산/초기 이식`과 `현장 OTA`를 둘 다 만족하도록 잡는다.

## 1. 최종 권장 구성

보드에는 아래 두 경로를 모두 남겨두는 것을 권장한다.

1. 평상시/현장 OTA용:
- `ESP32 TX -> ATmega RXD0`
- `ESP32 RX <- ATmega TXD0`
- `GND 공통`
- `ESP32 GPIO -> 트랜지스터 -> ATmega RESET`

2. 최초 1회 이식/복구용:
- `ISP 2x3 헤더`
- `MOSI / MISO / SCK / RESET / VCC / GND`

핵심은 이거다:

- 현장에서는 사실상 `UART + RESET`만 사용
- 하지만 처음 1회와 벽돌 복구를 위해 `ISP 헤더`는 남겨두는 게 가장 안전

---

## 2. 실제 권장 회로

```text
                          +---------------- ESP32-C5 ----------------+
                          |                                          |
                          | GPIO5  --------------------------------+ |--> ATmega RXD0
                          |                                         | |
                          | GPIO4  <-------------------------------+-+<-- ATmega TXD0
                          |                                          |
                          | GPIO24 ---[2.2k]---B   2N3904            |
                          |                       C----------------+  |
                          | GND -----------------E                |  |
                          |                                        |  |
                          | GPIO11 --------------------------------|--+--> ISP MOSI
                          | GPIO12 --------------------------------|----> ISP SCK
                          | GPIO9  <--- level shift / divider -----+<--- ISP MISO
                          | GND ---------------------------------------- GND
                          +----------------------------------------------+
                                                                 |
                                                                 |
                 +---------------------- ATmega128 ----------------------+
                 |                                                      |
                 | RXD0   <---------------------------------------------+--- ESP GPIO5
                 | TXD0   --------------------------------------------->+--- ESP GPIO4
                 | RESET  <------------------------------ 2N3904 C      |
                 | RESET ----[10k]---- +5V                             |
                 |                                                      |
                 | ISP MOSI <-------------------------------------------+--- ESP GPIO11
                 | ISP SCK  <-------------------------------------------+--- ESP GPIO12
                 | ISP MISO --------------------------------------------+--- ESP GPIO9
                 |                                                      |
                 | VCC ---- +5V                                         |
                 | GND ---- GND ----------------------------------------+--- ESP GND
                 +------------------------------------------------------+
```

---

## 3. UART OTA 필수 배선

이건 현장 OTA 때 실제로 쓰는 선이다.

| 기능 | ESP32-C5 | ATmega128 | 비고 |
|---|---|---|---|
| UART TX | `GPIO5` | `RXD0` | 기존 연결 유지 |
| UART RX | `GPIO4` | `TXD0` | 기존 연결 유지 |
| GND | `GND` | `GND` | 공통접지 필수 |
| RESET 제어 | `GPIO24` | `RESET` | 트랜지스터 통해 Low 제어 |

즉 네가 말한 방향대로, 현장 OTA용 실사용 회로는 거의 `TX/RX/GND + RESET`이 맞다.

---

## 4. RESET 제어 회로

`ATmega RESET`은 ESP GPIO에 직결하지 말고 NPN 트랜지스터로 제어하는 것을 권장한다.

```text
ESP32 GPIO24 ---[2.2k]---- Base (2N3904)
GND ---------------------- Emitter
Collector ---------------- ATmega RESET

ATmega RESET ---[10k]----- +5V
Base ---[100k]------------ GND   ; 선택사항
```

동작:

- `GPIO24 LOW` -> RESET 해제
- `GPIO24 HIGH` -> RESET LOW 강제

장점:

- 5V RESET 라인과 ESP 3.3V GPIO 직접 충돌 방지
- 자동 리셋 가능
- 부트로더 진입 타이밍 제어 쉬움

---

## 5. 최초 1회 이식용 ISP 헤더

최초에 부트로더와 퓨즈를 심기 위해 반드시 한 번은 필요하다.

권장 2x3 헤더:

```text
1  MISO
2  VCC
3  SCK
4  MOSI
5  RESET
6  GND
```

배선:

| ISP 핀 | ATmega128 | ESP32-C5와의 관계 |
|---|---|---|
| MISO | `MISO` | 향후 ESP `GPIO9`와도 연결 가능 |
| VCC | `+5V` | 외부 ISP 또는 시스템 전원 |
| SCK | `SCK` | 향후 ESP `GPIO12`와도 연결 가능 |
| MOSI | `MOSI` | 향후 ESP `GPIO11`와도 연결 가능 |
| RESET | `RESET` | 트랜지스터/외부 ISP 둘 다 접근 |
| GND | `GND` | 공통 |

추천:

- 현장 OTA만 생각해도 `ISP 헤더`는 없애지 말 것
- 생산/AS/복구 때 엄청 유용함

## 6. UART 레벨 주의

이 부분은 아주 중요하다.

현재 `GPIO4/5`로 이미 ATmega UART를 쓰고 있으니, 기존 하드웨어가 이미 검증됐다면 그대로 유지해도 된다.

하지만 원칙적으로는:

- `ESP32 TX 3.3V -> ATmega RXD0` : 대체로 직접 연결 가능
- `ATmega TXD0 5V -> ESP32 RX 3.3V` : 가능하면 레벨 다운 권장

만약 현재 보드에서 `ATmega TX -> ESP32 RX`가 이미 직결돼 있고 문제 없이 동작 중이면:

- 기존 회로를 유지하는 편이 현실적
- 새 리비전에서는 분압이나 레벨시프터를 넣는 것이 더 안전

권장 간단 보호:

```text
ATmega TXD0 ----[10k]----+----> ESP32 GPIO4
                         |
                        [20k]
                         |
                        GND
```

주의:

- 현재 보드가 이미 안정적으로 동작 중이라면, 기존 UART 라인은 함부로 바꾸지 않는 편이 낫다
- 새 보드 리비전에서만 반영하는 것이 안전하다

## 7. 추천 실제 배선안

### A안: 현재 보드 최대한 유지

- 기존 `TX/RX/GND` 그대로 사용
- `GPIO24 + 2N3904 + RESET`만 추가
- `ISP 2x3 헤더`만 추가

이게 가장 추천되는 1안이다.

이유:

- 기존 통신 회로를 거의 안 건드림
- 최초 1회 이식 가능
- 이후 UART OTA 가능
- 벽돌 시 ISP 복구 가능

### B안: 새 리비전 안전보강

- A안 전체 유지
- `ATmega TX -> ESP32 RX`에 레벨 다운 추가
- 필요하면 `ESP32/ATmega UART` 사이에 점퍼 또는 0옴 저항 추가

이건 다음 하드웨어 리비전용으로 더 좋다.

## 8. 가장 권장하는 부품
